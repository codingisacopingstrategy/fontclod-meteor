    Font-data is automatically saved to the MongoDB database while you edit the font. What you see above is a dump of all the data from the database, created with Meteor’s client-side Mongo adapter (JSON.stringify({ "glyphs" : Glyphs.find()._getRawObjects(), "meta" : Meta.find()._getRawObjects() })), you could try it in the console as well. A possible next step would be to create a way to serialize from and to UFO, and have the editor commit snapshots of the UFO to a Git(hub) repository.

    There is no straightforward way to load existing data at the moment: check out server/prepopulate.js for how the initial data is loaded. This could be changed to load from a JSON file or a webservice (GitHub repo) or a UFO on the filesystem etcetera (UFO from a Git repository over the GitHub API being the most promising for me…
